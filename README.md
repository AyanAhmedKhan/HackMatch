## 🚀 Quick Start

### **Prerequisites**

<details>
<summary>📋 System Requirements</summary>

| Tool | Version | Purpose |
|------|---------|---------|
| **Python** | `3.8+` | Backend development |
| **Node.js** | `14+` | Frontend tooling |
| **npm** | `6+` | Package management |
| **Git** | Latest | Version control |

```bash
# Verify installations
python --version    # Should show 3.8+
node --version      # Should show v14+
npm --version       # Should show 6+
```

</details>

### **Installation**

#### **1️⃣ Clone the Repository**
```bash
# Clone the project
git clone https://github.com/AyanAhmedKhan/HackMatch.git
cd HackMatch

# Verify project structure
ls -la
```

#### **2️⃣ Backend Setup**
```bash
# Navigate to backend directory
cd backend

# Create virtual environment (recommended)
python -m venv venv

# Activate virtual environment
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# Install Python dependencies
pip install -r requirements.txt

# Start backend server
python app.py
```

> 🔗 **Backend will run on**: `http://localhost:5000`

#### **3️⃣ Frontend Setup**
```bash
# Open new terminal and navigate to frontend
cd frontend

# Install frontend dependencies
npm install

# Start development server
npm start
```

> 🌐 **Frontend will run on**: `http://localhost:3000`

#### **4️⃣ Access the Application**
Open your browser and visit: **[http://localhost:3000](http://localhost:3000)**

## 🔧 Configuration

<details>
<summary>⚙️ Environment Setup</summary>

### **Backend Configuration**
Create a `.env` file in the `backend/` directory:

```env
# Flask Configuration
FLASK_ENV=development
FLASK_DEBUG=True
SECRET_KEY=your_secret_key_here

# API Keys
DEVPOST_API_KEY=your_devpost_api_key
EVENTBRITE_API_KEY=your_eventbrite_api_key
MLHACKS_API_KEY=your_mlhacks_api_key

# Database Configuration
DATABASE_URL=sqlite:///hackmatch.db

# External Service URLs
HACKATHON_API_BASE_URL=https://api.hackmatch.com/v1
CACHE_TIMEOUT=3600
```

### **Frontend Configuration**
Create a `config.js` file in the `frontend/` directory:

```javascript
const config = {
  API_BASE_URL: 'http://localhost:5000',
  APP_NAME: 'HackMatch',
  VERSION: '1.0.0',
  FEATURES: {
    DARK_MODE: true,
    NOTIFICATIONS: true,
    SOCIAL_LOGIN: false
  }
};
```

</details>

## 🚀 Deployment

<details>
<summary>☁️ Deploy to Vercel</summary>

### **Automatic Deployment**
1. **Fork** this repository to your GitHub account
2. **Connect** your GitHub account to Vercel
3. **Import** the HackMatch project
4. **Configure** environment variables in Vercel dashboard
5. **Deploy** with one click!

### **Manual Deployment**
```bash
# Install Vercel CLI
npm install -g vercel

# Deploy frontend
cd frontend
vercel --prod

# Deploy backend
cd ../backend
vercel --prod

# Link domains and configure
vercel domains add hackmatch.yourdomain.com
```

</details>

<details>
<summary>🐳 Docker Deployment</summary>

```dockerfile
# Dockerfile for full-stack deployment
FROM node:16-alpine as frontend
WORKDIR /app/frontend
COPY frontend/package*.json ./
RUN npm install
COPY frontend/ .
RUN npm run build

FROM python:3.9-slim as backend
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
COPY --from=frontend /app/frontend/dist ./static

EXPOSE 8000
CMD ["python", "app.py"]
```

```bash
# Build and run Docker container
docker build -t hackmatch .
docker run -p 8000:8000 -e DATABASE_URL=your_db_url hackmatch
```

</details>

### **📝 Contribution Guidelines**

1. **Fork** the repository and create your branch from `main`
2. **Install** dependencies and set up your development environment
3. **Make** your changes and test thoroughly

## 🐛 Troubleshooting

<details>
<summary>❗ Common Issues & Solutions</summary>

### **Backend Issues**

#### Port Already in Use
```bash
# Find process using port 5000
lsof -ti:5000 | xargs kill -9

# Or use different port
export FLASK_PORT=5001
python app.py
```

#### Python Dependencies Error
```bash
# Update pip and reinstall
pip install --upgrade pip
pip install -r requirements.txt --force-reinstall

# Create fresh virtual environment
rm -rf venv
python -m venv venv
source venv/bin/activate  # Mac/Linux
# or venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

### **Frontend Issues**

#### npm Install Failures
```bash
# Clear npm cache
npm cache clean --force

# Delete node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

#### CORS Errors
```javascript
// Add to your backend app.py
from flask_cors import CORS

app = Flask(__name__)
CORS(app, origins=['http://localhost:3000'])
```

### **Deployment Issues**

#### Vercel Build Failures
```bash
# Check build logs
vercel logs your-deployment-url

# Ensure all dependencies are listed
npm install --save missing-package
```

</details>
